Intructions:

What this is:

A hacked together janky string parser that lets you dictate how and where AP CSP-style blocks are drawn using only ASCII symbols.

What this is NOT:

This is NOT AP code parser, complier, validator or any other thing that might lead you to believe this is doing something beyond drawing pretty picutres on the screen.

This will NOT even convert AP text code into block form. Why? Because I'm too lazy to write a parser.

It will NOT help you write valid AP-style block pseudocode. You can draw it wrong using this tool.

What problem does it solve?

Writing the text-based form of AP pseduocode is easy. Drawing the blocks is anoying so this helps you automate it to some degree.

How do I use it?

Think of it this way: this a weird hybris a drawing tool. You type ASCII text in the input box and special characters are interpreted to draw AP-style blocks.

For now only 5.5 things are recognized. Here's the list:

  1. [code] -- Rounded Rectangle - used by most ap-style statements. e.g. [a <- 7]
  2. (condition) -- Oval - used by conditions and conditional expressions
  3. |value| -- Square Rectangle - used for arguments and parameters
  4. <- -- Converts to left arrow symbol: ←
  5. (2 spaces) -- use for indenting. Single spaces are ignored.
Half-thing: making blocks gray
  • IF, REPEAT, PROCEDURE, FOR -- The interpreter will recognize these keywords and make the block be shaded in gray. You don't have to do anything to make the blocks gray. It is done auto-magically.
